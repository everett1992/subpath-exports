vitest and tsx don't resolve subpath imports, exports, or project references to the typescript source.
Instead they rely on compiled JavaScript in the outDir.

This project has valid TypeScript, it compiles with `tsc` and tsserver provides type hints and definitions for the imports.
However tsx and vitest both fail to resolve the subpath self reference imports unless the source is compiled to JavaScript.

```console
$ npx tsc
<success>
```


```console
$ npx tsx src/index.ts
[
  {
    status: 'fulfilled',
    value: [Module: null prototype] { default: Symbol(export) }
  },
  {
    status: 'rejected',
    reason: Error [ERR_MODULE_NOT_FOUND]: Cannot find module 'subpath-exports/dist/sub.js' imported from subpath-exports/src/index.ts
      code: 'ERR_MODULE_NOT_FOUND'
    }
  },
  {
    status: 'rejected',
    reason: Error [ERR_MODULE_NOT_FOUND]: Cannot find module subpath-exports/dist/sub.js' imported from subpath-exports/src/index.ts
      code: 'ERR_MODULE_NOT_FOUND'
    }
  }
]

```
